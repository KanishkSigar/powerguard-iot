# Contributing to PowerGuard IoT

First off, thank you for considering contributing to PowerGuard IoT! It's people like you that make open-source a great community.

## How to Contribute

1. **Fork the Repository**: Start by forking the repository to your own GitHub account.
2. **Clone the Repo**: Clone it to your local machine.
3. **Create a Branch**: Create a new branch for your feature or bug fix (`git checkout -b feature/my-new-feature`).
4. **Make Changes**: Implement your feature or bug fix.
5. **Commit your Changes**: Write clear, descriptive commit messages. We prefer conventional commits (e.g., `feat: add new dashboard widget`).
6. **Push to your Fork**: Push your changes to your fork on GitHub.
7. **Submit a Pull Request**: Open a pull request against the `master` branch of this repository.

## Development Environment Setup

Please refer to the `README.md` for detailed instructions on how to set up the backend, frontend, and database locally using Docker Compose and Python.

## Reporting Bugs

If you find a bug, please open an issue on GitHub. Include as much detail as possible:
- Steps to reproduce the bug
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- OS and browser versions

## Feature Requests

We welcome feature requests! Please open an issue describing the feature, why it would be useful, and any potential implementation ideas you might have.

Thank you for contributing!
